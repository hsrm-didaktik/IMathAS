Macro Library js_utils

Utilities using javascript to save data on client without writing javascript code in question control or text. This can be used, for instance, to save information on the state of an ongoing assessment.

Version 1.0 September 23, 2025

>

Types of Storage

This library supports saving and retrieving 3 data on the client side in 3 ways:

  • As a cookie: Cookies are short strings that are saved and retrieved under a key. Cookies are set when the question text is displayed. They are retrieved in the question control when the question is generated on the server. Cookies set using this library persist for the time of the session.
  • In session storage: Session storage items can be longer than cookies. They are saved and retrieved under a key. Session storage items are set and retrieved in the browser and therefore they must be embedded into the question text. Session storage items are tied to a particular window. Therefore, questions opened in a new window, e.g. on preview, will not see any previously set session storage items, but questions within a test window can see such items set previously by other questions in the same test. Questions may show the content of session storage items when displayed, but there is no way to use such items while the question is generated. Question storage items persist as long as the respective window exists.
  • In local storage: Local storage items can be longer than cookies. They are saved and retrieved under a key. Local storage items are set and retrieved in the browser and therefore they must be embedded into the question text. Questions may show the content of local storage items when displayed, but there is no way to use such items while the question is generated. local storage items persist until they are explicitely deleted. Therefore it is important to provide users with the possibility to clear the local storage using the button generated by the respective macro.

Cookies

js_setCookie($key,$value)

sets the cookie with key $key and value $value. Returned value must be placed in question text.

getCookie($key)

gets the value of the cookie with key $key if it exists, NULL otherwise. Must be placed in question control.

Session Storage

js_setSession($key,$vakue)

sets the session key $key with value $value. $value must be a string which may contain HTML markup. Returned value must be placed in question text.

js_getSession($key)

gets the value of session key $key if it exists, null otherwise, and prints it. Must be placed in question text

Local Storage

js_setLocal($key,$vakue)

sets the loxal storage key $key with value $value. $value must be a string which may contain HTML markup. Returned value must be placed in question text.

js_getLocal($key)

gets the value of local storage key $key if it exists, null otherwise, and prints it. Must be placed in question text

js_clearLocal([$label,$confirmation])

shows a button with label $label. Clicking the button clears the local storage. When clicked, $confirmation is shown to confirm the action. Returned value must be placed in question text.
